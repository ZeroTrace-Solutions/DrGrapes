# DrGrapes Email Templates

These templates were created using the design tokens extracted from both `student-mobile-app` and `web-app` projects. They adapt the overall brand aesthetic (dark theme, vibrant magenta accents, Plus Jakarta Sans typography) into a format optimised for HTML emails.

## Template Overview

- `welcome.html`: Welcome onboarding email with a CTA button to get started.
- `verify-email.html`: Verification code email (using the same OTP format as the backend).
- `reset-password.html`: Password reset code email matching the authentication logic.
- `notification.html`: Generic notification template with a message content area and optional CTA.
- `account-update.html`: Informational email to confirm successful account changes (e.g., password/email change).

## Theme Extraction & Adjustments

The tokens were derived primarily from the mobile and web `global.css` / `index.css` files, noting the consistent dark palette and typography.

- **Background**: `#121414` (Surface/Background)
- **Container Background**: `#1e2020` (Surface Container)
- **Primary Accent**: `#c13584` (Vibrant Magenta - used for buttons and main accents)
- **Secondary Accent**: `#0d36c3` (Royal Blue - used in notifications)
- **Highlight Text**: `#ffafd2`
- **Main Text**: `#e2e2e2`, `#dcbfc9` (Muted)
- **Borders**: `#282a2b` around structural elements, `#564149` around specific containers.
- **Font Stack**: `'Plus Jakarta Sans', Helvetica, Arial, sans-serif`. (Since Plus Jakarta Sans is a web font, email clients that strip external `@import` tags will gracefully fall back to Helvetica/Arial).

_Note: Since the brand uses a dark aesthetic, we provided these emails in Dark Mode by default. Most modern clients handle inline dark mode well, provided we explicitly set text colors and backgrounds._

## Template Variables

The templates use `{{variableName}}` syntax. Before sending via `MailService`, replace these strings with the actual data:

- `{{firstName}}`: User's first name
- `{{preheader}}`: Hidden text shown in the email client inbox preview
- `{{actionUrl}}`: The URL for CTA buttons
- `{{actionText}}`: Text for the CTA button in the generic notification
- `{{otp}}`: The one-time password string generated by `generateAndSendOTP`
- `{{updateType}}`: E.g., "password" or "email address"
- `{{notificationTitle}}`: Heading for notifications
- `{{notificationContent}}`: Body text for notifications
- `{{year}}`: Current year for footer copyright

### System URLs (Configure via `.env`)

All hyperlinks have been replaced with variables to be easily updated from your environment variables:

- `{{baseUrl}}`: Link to the main platform (e.g., `process.env.FRONTEND_URL`)
- `{{supportUrl}}`: Link to support/contact page
- `{{privacyUrl}}`: Link to Privacy Policy
- `{{termsUrl}}`: Link to Terms of Service
- `{{unsubscribeUrl}}`: Link to the unsubscribe handler
- `{{addressUrl}}`: Link to company address map/info
- `{{notificationsUrl}}`: Link to push/email preferences update

## Usage Instructions

1. **Placeholders**: Replace the `<img>` tags pointing to `via.placeholder.com` with absolute URLs to your actual hosted brand assets (e.g., `https://drgrapes.com/assets/logo.png`).
2. **Inlining CSS**: The provided templates contain internal `<style>` tags tailored for mobile responsiveness and hover effects. However, inline CSS on the HTML nodes themselves has been pre-applied for maximum client compatibility. You can pass the raw HTML output as a string directly to `nodemailer`. If you add new styles to the `<head>`, consider using a tool like [Juice](https://github.com/Automattic/juice) in your build step to inline the CSS.
3. **Integration**: Within `backend/src/common/mail/mail.service.ts`, import the file system (`fs`) module, read the desired template, `.replace('{{otp}}', otp)`, and pass the resulting string to the `html` field (instead of the `text` field) of `sendMail()`.
